EchoDB: Technical Specification

Project: Advanced Data Structures & Algorithms Mini-Project

Architecture: Log-Structured Merge-Tree (LSM) with Adaptive Indexing

1. Executive Summary

EchoDB is a high-performance, embedded key-value storage engine designed from first principles. It prioritizes write-throughput and adaptive data retrieval using self-adjusting data structures. Unlike general-purpose engines, EchoDB implements specific algorithms from the ADSA syllabus to optimize for skewed workloads and efficient storage.

2. Technical Architecture

A. Write Path (Durability Level: High)

  1. Write-Ahead Log (WAL): Append-only sequential disk write for crash recovery.
  2. MemTable (In-Memory): Data is inserted into a Splay Tree. Amortized O(log n). Frequently accessed keys migrate to the root.
  3. Flushing: Threshold-based transition from RAM to Disk.

B. Read Path (Latency Level: Low)

  1. Memory Search: Initial lookup in the Splay Tree.
  2. Disk Search (SSTables): Multi-layered search in sorted files.
  3. Internal Indexing: Each SSTable block is indexed by a B+Tree at the footer to minimize disk seek operations.

4. Key Differentiators

COMPARATIVE ANALYSIS

EchoDB vs. Industry Standards (LevelDB/RocksDB):

  • Splay Tree vs. SkipList: Standard engines use randomized SkipLists. EchoDB uses Splay Trees to physically reorganize bits based on access frequency, providing better locality for "hot" keys.
  • Fused Compression: Huffman tables are recalculated during every compaction cycle, optimizing the dictionary for the specific data currently in the database.
  • Hybrid Priority: Fibonacci Heaps are used to solve the multi-objective problem of which files to merge (balancing file age vs. deletion volume).
